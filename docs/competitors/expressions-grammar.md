# Различия синтаксиса выражений C++ и Pascal

## 1. Сравнение числовых типов данных:

### C++

* `int` - целочисленные значения;
* `short int` и `long int` — разновидности целочисленных типов с разным размером памяти;
* `float` и `double` — числа с плавающей точкой (double обычно имеет большую точность и размер, чем float);
* `long double` — расширенный тип данных для чисел с плавающей точкой;
* `bool` — логический тип (`true/false`);
* также поддерживаются беззнаковые целочисленные типы (например, `unsigned int`).
* `char`, `signed char`, `unsigned char` - cимвольные 8-битные целые числа.

### Pascal]()

* `integer` — целочисленные значения;
* `smallint`, `longint`, `int64` — разновидности целочисленных типов с разным размером памяти;
* `real` — вещественные числа (в некоторых диалектах могут быть дополнительные типы, например, `single`, `double`,
  `extended`);
* `byte`, `word` - беззнаковые целые числа
* `boolean` — логический тип (`true/false`);

### Вывод: C++ более гибкий (любой `int` может быть `unsigned`) и рассматривает

`char` как число. Pascal строже: беззнаковые типы — это отдельные именованные типы (
`byte, word`), а не модификация существующих.

## 2. Сравнение арифметических операторов:

| Операция                   | C++                          | Pascal    | Примечание                                                                                                                                                                                                               |
|:---------------------------|:-----------------------------|:----------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Присваивание**           | **`=`**                      | **`:=`**  | **Фундаментальное различие.** В C++ `=` — это оператор-выражение. В Pascal `:=` — это *инструкция*, а не выражение.<br/>То есть в C++ можно написать a = b = c = 10, то в Pascal не допускается писать a := b := c := 10 |
| **Сложение**               | `+`                          | `+`       |                                                                                                                                                                                                                          |
| **Вычитание**              | `-`                          | `-`       |                                                                                                                                                                                                                          |
| **Умножение**              | `*`                          | `*`       |                                                                                                                                                                                                                          |
| **Деление (Float)**        | **`/`**                      | **`/`**   | В C++ `/` выполняет целочисленное деление, если *оба* операнда целые. В Pascal `/` *всегда* возвращает `Real`.                                                                                                           |
| **Деление (Int)**          | **`/`**                      | **`div`** | В C++ для целочисленного деления используется тот же `/`. Pascal требует явный оператор `div`.                                                                                                                           |
| **Остаток**                | **`%`**                      | **`mod`** | Символ `%` против ключевого слова `mod`.                                                                                                                                                                                 |
| **Инкремент**              | `++`                         | (нет)     | В C++ есть `++x` (префикс) и `x++` (постфикс). В Pascal используется процедура `Inc(x)`.                                                                                                                                 |
| **Декремент**              | `--`                         | (нет)     | Аналогично, `Inc(x, -1)` или `Dec(x)`.                                                                                                                                                                                   |
| **Составное присваивание** | `+=`, `-=`, `*=`, `/=`, `%=` | (нет)     | C++ позволяет `x += 5;`. Pascal требует полной записи: `x := x + 5;`.                                                                                                                                                    |

* в C++ также есть побитовые операторы (&, |, ^, <<, >>), которые в Pascal реализуются через другие средства или
  функции.
  **Итог:** C++ имеет более богатый набор операторов (`++`, `+=`), которые также являются выражениями. Pascal использует
  ключевые слова (`div`, `mod`) и разделяет операцию деления на `div` (для `Integer`) и `/` (для `Real`).

## 3. Сравнение приоритетов операций

**C++ (упрощенно):**

1. `()` `[]` `->` `.` `++` `--` (высший)
2. `!` `~` `+` `-` `*` `&` (унарные)
3. `*` `/` `%` (мультипликативные)
4. `+` `-` (аддитивные)
5. `<<` `>>` (сдвиги)
6. `<` `<=` `>` `>=` (отношения)
7. `==` `!=` (равенство)
8. `&` (побитовое И)
9. `^` (побитовое XOR)
10. `|` (побитовое ИЛИ)
11. **`&&` (логическое И)**
12. **`||` (логическое ИЛИ)**
13. `?:` (тернарный)
14. `=` `+=` `-=` (присваивание) (низший)

**Pascal (полностью):**

1. `@` `not` (высший)
2. `*` `/` `div` `mod` **`and`** (мультипликативные)
3. `+` `-` **`or`** `xor` (аддитивные)
4. `=` `<>` `<` `>` `<=` `>=` `in` (отношения) (низший)

**Критическое различие:**
В C++ логические операторы (`&&`, `||`) имеют **очень низкий** приоритет.
В Pascal логические операторы (`and`, `or`) имеют **тот же приоритет, что и умножение/сложение**.

Из-за этого простое выражение C++:
`x > 0 && y < 10`
В Pascal **вызовет ошибку компиляции**, если написать его так:
`x > 0 and y < 10`
Компилятор Pascal видит это так: `x > (0 and y) < 10` (пытается применить `and` к числу `0` и переменной `y`).
Правильный синтаксис в Pascal **требует скобок**:
`(x > 0) and (y < 10)`

**Итог:** У C++ сложная, но гибкая система приоритетов. У Pascal — очень простая, но требующая частого использования
скобок в логических выражениях.

## 4\. Список лево- и правоассоциативных операторов

Ассоциативность определяет, как выполняются операторы с одинаковым приоритетом (слева направо или справа налево).

* **C++:**

    * **Левоассоциативные** (большинство):
        * `*` `/` `%`
        * `+` `-`
        * `<<` `>>`
        * `&&` `||`
        * `a - b + c` парсится как `(a - b) + c`
    * **Правоассоциативные** (меньшинство, но важные):
        * **Все операторы присваивания** (`=` `+=` `*=`). Это позволяет писать `a = b = c;` (парсится как
          `a = (b = c);`).
        * Тернарный оператор `?:`.
        * Все *унарные* операторы (`-x`, `!x`, `*ptr`, `&var`, `++x`).

* **Pascal:**

    * **Левоассоциативные:**
        * Все бинарные операторы: `* / div mod and`, `+ - or xor`.
        * `a - b + c` парсится как `(a - b) + c`.
    * **Правоассоциативные:** (Нет).
    * **Неприменимо (N/A):**
        * **Присваивание `:=` не является оператором-выражением**. Это инструкция. У нее нет ни приоритета, ни
          ассоциативности.
        * Запись `a := b := c;` **недопустима синтаксически**.

**Итог:** В C++ присваивание правоассоциативно и позволяет "цепочки". В Pascal присваивание — это инструкция, и цепочки
невозможны.

## 5\. Сравнение встроенных функций для работы с числами

* **C++:**

    * В самом языке функций почти нет. Все они подключаются через заголовочные файлы, в основном **`<cmath>`** (для
      математики) и **`<cstdlib>`** (для `abs` для `int`).
    * `sqrt(d)`, `pow(d, p)` (возведение в степень).
    * `sin(d)`, `cos(d)`, `tan(d)`.
    * `floor(d)` (округление вниз), `ceil(d)` (округление вверх).
    * `fmod(x, y)` (остаток для `double`).
    * `abs(i)` (из `<cstdlib>`) или `std::abs(f)` (перегруженная версия из `<cmath>`).

* **Pascal:**

    * Многие функции встроены в язык (или автоматически подключаемый модуль `System`).
    * `Sqrt(x)`, **`Sqr(x)`** (возведение в квадрат — очень частая функция).
    * `Sin(x)`, `Cos(x)`, `ArcTan(x)`.
    * **`Trunc(x)`** (отбросить дробную часть, вернуть `Integer`).
    * **`Round(x)`** (математическое округление, вернуть `Integer`).
    * **`Frac(x)`** (вернуть дробную часть).
    * `Abs(x)`.
    * *Процедуры*: `Inc(x)` (увеличить `x` на 1), `Dec(x)` (уменьшить `x` на 1).

**Итог:** В Pascal больше встроенных функций для базовой математики и преобразований типов (особенно `Round` и `Trunc`),
которые ощущаются как часть языка. C++ полагается на стандартную библиотеку `<cmath>`.

## 6\. Прочие особенности синтаксиса выражений

1. **Присваивание как выражение (C++):**

* В C++ `a = b` — это выражение, которое *имеет значение* (значение `b`).
* Это позволяет: `while ((c = getchar()) != EOF) { ... }`
* Это также источник ошибок: `if (x = 5)` (случайно присвоили, вместо `if (x == 5)`).

2. **Присваивание как инструкция (Pascal):**

* В Pascal `a := b` — это инструкция (команда). Она *не имеет значения*.
* Ее нельзя использовать внутри другого выражения: `if x := 5 then ...` — это **синтаксическая ошибка**. Это делает код
  строже и безопаснее.

3. **Тернарный оператор (C++):**

* C++ имеет условное выражение: `result = (a > b) ? a : b;`
* Pascal такого оператора не имеет. Требуется полная инструкция:
  ```pascal
  if a > b then
    result := a
  else
    result := b;
  ```
* (Современные диалекты, вроде Free Pascal, добавили функцию `IfThen(cond, a, b)`).

4. **Точка с запятой (`;`):**

* **C++:** Точка с запятой — это **терминатор инструкции**. Каждая инструкция (включая присваивание) *должна*
  заканчиваться `;`.
* **Pascal:** Точка с запятой — это **разделитель инструкций**. Она ставится *между* инструкциями, но не обязательна
  после последней инструкции в блоке (`end`).
